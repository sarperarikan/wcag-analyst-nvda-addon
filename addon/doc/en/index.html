WCAG Analyst - User Guide

WCAG Analyst is an advanced NVDA add-on that analyzes web page elements and HTML content for WCAG accessibility compliance using Ollama AI models. It provides detailed reports with improvement suggestions in English and Turkish.

1. Requirements

  • NVDA 2023.1 or later
  • Ollama installed and running on your system
  • At least one Ollama model downloaded (e.g., llama3.2, mistral)
Note: To download an Ollama model, open a command prompt and run:
ollama pull llama3.2

2. Installation

  1. Download the WCAG Analyst add-on file (.nvda-addon)
  2. Double-click the file to install it in NVDA
  3. Restart NVDA
  4. Make sure Ollama is running on your computer

3. Features

  • HTML Analyst Interface: Two-panel interface to input HTML and view analysis results
  • Element Analysis: Analyze the focused web element with a single shortcut
  • Ask More: Ask follow-up questions about the analysis results
  • WCAG 2.0/2.1/2.2 Support: Analysis based on different WCAG versions
  • A/AA/AAA Levels: Conformance level selection
  • Multi-language: Turkish and English support

4. Quick Start

  1. Install the WCAG Analyst add-on in NVDA
  2. Make sure Ollama is running on your computer
  3. Navigate to any web page element
  4. Press the keyboard shortcut to analyze:

NVDA + Shift + W

Or use "WCAG HTML Analyst" from the NVDA Tools menu.

5. Menu Items

The following items are available in the NVDA Tools menu:

Menu Item Description
WCAG HTML Analyst Opens the two-panel HTML analysis interface
WCAG Element Analysis Analyzes the focused element (same as NVDA+Shift+W)
WCAG Analyst Settings Opens the add-on settings panel
WCAG Analyst Documentation Opens this documentation page

6. HTML Analyst Interface

HTML Analyst provides a modern two-panel interface:

Left Panel - HTML Content

  • Paste the HTML code to be analyzed here
  • Click "Analyze" button to start the analysis

Right Panel - Improvements and Suggestions

  • Read-only area - displays analysis results
  • WCAG compliance issues and recommendations
  • Corrected code examples

Ask More

  • Ask follow-up questions about the analysis results
  • When a question is asked, the response replaces the previous content
  • Conversation history is preserved, allowing contextual questions

7. Configuration

Access settings: NVDA Menu → Preferences → Settings → WCAG Reporter or "WCAG Analyst Settings" from the Tools menu

Ollama Server Settings

Setting Description Default
Server URL Ollama server address http://localhost:11434
Model AI model to use for analysis llama3.2

WCAG Analysis Settings

Setting Description Default
WCAG Version Which WCAG version to check against 2.2
Conformance Level A, AA, or AAA AA
Language Analysis result language Turkish
Timeout Maximum wait time for AI response 120 seconds

8. Keyboard Shortcuts

Shortcut Action
NVDA + Shift + W Analyze the focused element

9. Result Window

The analysis result window is fully accessible with screen readers:

  • Tab - Navigate between sections and buttons
  • Copy - Copy analysis result to clipboard
  • Ask - Ask a follow-up question
  • Escape or Close - Close the window

10. Troubleshooting

Connection Error

If you see "Connection failed":
  1. Make sure Ollama is running: Open command prompt and type ollama serve
  2. Check the server URL in settings (default: http://localhost:11434)
  3. Use the Refresh Models button to update the model list

No Models

If the model list is empty:

  1. Open command prompt
  2. Run: ollama pull llama3.2
  3. Click "Refresh Models" button in settings

Analysis Takes Too Long

Large or complex elements may take longer. Try:

  • Use a faster model (e.g., llama3.2:1b)
  • Increase the timeout in settings
  • Analyze simpler elements first
