WCAG Analyst - Kullanım Kılavuzu

WCAG Analyst, Ollama AI modellerini kullanarak web sayfası elementlerini ve HTML içeriklerini WCAG erişilebilirlik uyumluluğu açısından analiz eden gelişmiş bir NVDA eklentisidir. Türkçe ve İngilizce dillerinde iyileştirme önerileri içeren detaylı raporlar sunar.

1. Gereksinimler

  • NVDA 2023.1 veya üstü
  • Sisteminizde kurulu ve çalışır durumda Ollama
  • En az bir Ollama modeli indirilmiş olmalı (örn: llama3.2, mistral)
Not: Ollama modeli yüklemek için komut istemcisini açın ve şu komutu çalıştırın:
ollama pull llama3.2

2. Kurulum

  1. WCAG Analyst eklenti dosyasını (.nvda-addon) indirin
  2. Dosyayı çift tıklayarak NVDA'ya yükleyin
  3. NVDA'yı yeniden başlatın
  4. Ollama'nın bilgisayarınızda çalıştığından emin olun

3. Özellikler

  • HTML Analyst Arayüzü: İki panelli arayüz ile HTML içeriği girip analiz sonuçlarını görün
  • Element Analizi: Odaktaki web elementini tek kısayolla analiz edin
  • Daha Sor: Analiz sonuçları hakkında takip soruları sorun
  • WCAG 2.0/2.1/2.2 Desteği: Farklı WCAG sürümlerine göre analiz
  • A/AA/AAA Seviyeleri: Uyumluluk seviyesi seçimi
  • Çoklu Dil: Türkçe ve İngilizce destek

4. Hızlı Başlangıç

  1. WCAG Analyst eklentisini NVDA'ya yükleyin
  2. Ollama'nın bilgisayarınızda çalıştığından emin olun
  3. Herhangi bir web sayfası elementine gidin
  4. Analiz için klavye kısayoluna basın:

NVDA + Shift + W

Veya NVDA Araçlar menüsünden "WCAG HTML Analyst" seçeneğini kullanın.

5. Menü Öğeleri

NVDA Araçlar menüsünde aşağıdaki öğeler bulunur:

Menü Öğesi Açıklama
WCAG HTML Analyst İki panelli HTML analiz arayüzünü açar
WCAG Element Analizi Odaktaki elementi analiz eder (NVDA+Shift+W ile aynı)
WCAG Analyst Ayarları Eklenti ayarlar panelini açar
WCAG Analyst Dokümantasyon Bu dokümantasyon sayfasını açar

6. HTML Analyst Arayüzü

HTML Analyst, iki panelli modern bir arayüz sunar:

Sol Panel - HTML İçeriği

  • Analiz edilecek HTML kodunu buraya yapıştırın
  • "Analiz Et" butonuna tıklayarak analizi başlatın

Sağ Panel - İyileştirmeler ve Öneriler

  • Salt okunur alan - analiz sonuçlarını gösterir
  • WCAG uyumluluk sorunları ve önerileri
  • Düzeltilmiş kod örnekleri

Daha Sor

  • Analiz sonuçları hakkında takip soruları sorun
  • Soru sorulduğunda yanıt, önceki içeriğin yerine gösterilir
  • Konuşma geçmişi korunur, böylece bağlamsal sorular sorabilirsiniz

7. Yapılandırma

Ayarlara erişim: NVDA Menüsü → Tercihler → Ayarlar → WCAG Reporter veya Araçlar menüsünden "WCAG Analyst Ayarları"

Kısayol Eylem

9. Sonuç Penceresi

Analiz sonucu penceresi ekran okuyucularla tamamen erişilebilirdir:

  • Tab - Bölümler ve butonlar arasında gezin
  • Kopyala - Analiz sonucunu panoya kopyala
  • Sor - Takip sorusu sorun
  • Escape veya Kapat - Pencereyi kapat

10. Sorun Giderme

Bağlantı Hatası

"Bağlantı başarısız" görüyorsanız:
  1. Ollama'nın çalıştığından emin olun: Komut istemcisini açın ve ollama serve yazın
  2. Ayarlardaki sunucu URL'sini kontrol edin (varsayılan: http://localhost:11434)
  3. Modelleri Yenile butonunu kullanarak model listesini güncelleyin

Model Yok

Model listesi boşsa:

  1. Komut istemcisini açın
  2. Çalıştırın: ollama pull llama3.2
  3. Ayarlarda "Modelleri Yenile" butonuna tıklayın

Analiz Çok Uzun Sürüyor

Büyük veya karmaşık elementler daha uzun sürebilir. Şunları deneyin:

  • Daha hızlı bir model kullanın (örn: llama3.2:1b)
  • Ayarlarda zaman aşımını artırın
  • Önce daha basit elementleri analiz edin
